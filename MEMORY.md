## 二、记忆结构

当前采用轻量三层结构：

### 1. `NOW.md`
用于记录当前状态：
- 现在最重要的事
- 正在推进的事项
- 阻塞与风险
- 下一步动作

这是当前工作台，允许频繁刷新。

### 2. `memory/YYYY-MM-DD.md`
用于记录每天发生的原始事件：
- 今天做了什么
- 做了哪些决定
- 有什么临时上下文
- 哪些事项需要后续跟进

这是 daily log，偏原始，允许粗糙，但要真实。

### 3. `knowledge/*.md`
用于记录长期有效、可复用的知识：
- 偏好
- 决策
- 流程
- 项目背景

这是提炼层，不写流水账。

---

## 三、写入规则

### 写进 `NOW.md`，如果：
- 它属于当前状态
- 它影响今天的优先级
- 它是正在推进中的事项
- 它是当前阻塞或下一步

### 写进 daily memory， 如果：
- 这是今天发生的事
- 这还只是上下文，未提炼
- 这是一个操作记录、发现、临时判断

### 写进 `knowledge/*.md`，如果：
- 下周、下月仍会有用
- 它会影响未来协作方式
- 它是稳定偏好、决策、规则、流程或项目背景

---

## 四、长期协作原则

### 沟通偏好
- 默认中文
- 需要时可处理中英日三语
- 喜欢清晰、专业、可执行的表达
- 希望先给结论，再给细节

### 协作偏好
- 希望助手主动，但不要太吵
- 希望先读上下文、先判断、再提问
- 希望重复工作被整理成模板、流程或工具
- 希望复杂问题被拆解后再执行

### 风险规则
- 外部发送、公开发布、真实业务动作前先确认
- 修改密钥、账号、生产配置前先确认
- 花钱或可能产生实际业务影响的动作前先确认
- 不记录敏感密钥，不泄露隐私

---

## 五、使用原则

- 文件是事实来源，不依赖会话天然记忆
- 当前状态看 `NOW.md`
- 最近上下文看 daily memory
- 长期知识看 `knowledge/`
- `MEMORY.md` 只保留总览、原则和入口

---

## 六、记忆铁律

- 重要信息不要只留在聊天里
- 要记住，就写进文件
- 当前态放 `NOW.md`
- 原始日志放 daily memory
- 提炼知识放 `knowledge/`
- `MEMORY.md` 只做总览与导航
- 宁可短而清楚，不要堆废话
